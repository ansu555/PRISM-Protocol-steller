# SDK & Bindings

PRISM Core is a [Soroban](https://developers.stellar.org/docs/build/smart-contracts) smart contract on Stellar. You integrate it from TypeScript with two pieces:

1. **`@stellar/stellar-sdk`** — the official Stellar SDK. It builds, simulates, signs, and submits Soroban transactions and decodes contract return values.
2. **Generated contract bindings** (optional) — a typed client the Stellar CLI produces directly from the deployed contract, so every function is fully typed and can never drift from the on-chain binary.

There is no separate SDK package to install. A Soroban contract publishes its interface spec on-chain, and the Stellar CLI reads that spec directly to generate a typed client.

## Install

<CodeGroup>
```bash npm
npm install @stellar/stellar-sdk
```

```bash pnpm
pnpm add @stellar/stellar-sdk
```

```bash bun
bun add @stellar/stellar-sdk
```
</CodeGroup>

PRISM is built and tested against `@stellar/stellar-sdk` v15.

## Generate typed bindings

The [Stellar CLI](https://developers.stellar.org/docs/tools/cli) reads the deployed contract spec and emits a typed TypeScript client — no hand-written types, no copy step:

```bash
stellar contract bindings typescript \
  --network testnet \
  --id CCULBBT4PA64GWXSKT4G7HOYQ4RXRNYY2JP5MZ2G73VKRFAJ6CHB3RZK \
  --output-dir ./src/bindings/prism-core
```

Regenerate whenever the contract is redeployed. The generated client wraps every public function with correct argument and return types.

## Deployed addresses

Import contract IDs from a single module instead of hard-coding them inline:

```ts
// addresses.ts
export const PRISM_CORE   = 'CCULBBT4PA64GWXSKT4G7HOYQ4RXRNYY2JP5MZ2G73VKRFAJ6CHB3RZK';
```

## The TrancheKind enum

Every tranche-scoped call takes a `kind` (a `u32` index). It maps to the on-chain `TrancheKind` enum:

```ts
export enum TrancheKind {
  Prime = 0, // 5% target  · paid first  · loss-protected
  Core  = 1, // 8% target  · intermediate
  Alpha = 2, // 15% target · first-loss absorber
}
```

## Amounts are `i128`

USDC has **7 decimals** on Stellar. Every amount crosses the contract boundary as an `i128` — always use `BigInt`, never a JavaScript `number`:

```ts
import { nativeToScVal } from '@stellar/stellar-sdk';

const oneUsdc = 10_000_000n;                       // 1 USDC = 1e7 stroops
const amount  = nativeToScVal(oneUsdc, { type: 'i128' });
```

A tranche's NAV per share is stored on the `Tranche` struct as `nav_per_share_q`, a Q64.64 fixed-point `i128` where `1.0 == 2^64`. Read it with `get_tranche` and divide by `2n ** 64n` for a human-readable ratio — there is no separate `get_nav` call.

## Error codes

`prism_core` raises a typed `PrismError`. The numeric codes are stable across the contract ABI:

| Code | Variant | Meaning |
| --- | --- | --- |
| 1 | `VaultNotActive` | Vault is not in the `Active` state |
| 2 | `VaultPaused` | Protocol is paused; user operation blocked |
| 3 | `InvalidTrancheKind` | `kind` is not 0, 1, or 2 |
| 4 | `LoanInWrongState` | Loan operation invalid for its current state |
| 5 | `InsufficientLiquidity` | Not enough reserve to satisfy the request |
| 6 | `SlippageExceeded` | Output below the supplied minimum |
| 7 | `Unauthorized` | Caller is not the admin / required signer |
| 10 | `ArithmeticOverflow` | Checked arithmetic overflowed |
| 11 | `EmptyTrancheNav` | NAV undefined (no supply) |
| 12 | `InvalidSeverity` | `severity_bps` or `event_type` out of range |
| 13 | `LossExceedsTotalAssets` | Loss larger than total vault assets |
| 14 | `TrancheWipedNoDepositsAllowed` | Deposit blocked into a wiped tranche (NAV 0, supply > 0) |
| 20 | `BorrowerMismatch` | Signer is not the loan's borrower |
| 30–33 | `Encrypt*` | Encrypt FHE attestation errors |
| 34 | `OracleNotAllowlisted` | Signing key is not in the oracle allowlist |
| 35 | `OracleAllowlistFull` | Allowlist already holds 8 keys |
| 36 | `OracleAlreadyAllowlisted` | Key already present |
| 40–43 | `Cloak*` | Cloak batch-payout attestation errors |
| 50 | `AlreadyInitialized` | `init_*` called twice |
| 51 | `NotInitialized` | Contract / vault / tranche used before setup |
| 60–65 | `Collateral*` | Collateral attestation lifecycle errors |

When a transaction fails, the Soroban RPC error carries the contract error code — map it back to the variant above to surface a precise message in your UI.
