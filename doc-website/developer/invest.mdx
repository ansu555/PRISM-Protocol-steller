# Deposit into a Tranche

This walkthrough deposits USDC into the **Core** tranche of vault `0`, reads the resulting position, and withdraws. It uses the `server`, `contract`, and `signer` from [Connect a Client](/developer/connect).

## 1. Hold USDC

PRISM settles in USDC. On Stellar, an account must hold a **trustline** to an asset before it can receive it. The frontend does this for the user; from a script you add it once:

```ts
import { Operation, Asset, TransactionBuilder, BASE_FEE, Networks } from '@stellar/stellar-sdk';

// Testnet PRISM uses a mintable test USDC (code PTUSDC) so demos are fully scriptable.
const usdc = new Asset('PTUSDC', 'GCZFPAJEJHMQPZ4BQUWUEBV7KJQ7GEKDF4FAWYUW4NOIRSWXCMDEOESW');

const account = await server.getAccount(signer.publicKey());
const tx = new TransactionBuilder(account, { fee: BASE_FEE, networkPassphrase: Networks.TESTNET })
  .addOperation(Operation.changeTrust({ asset: usdc }))
  .setTimeout(30)
  .build();
tx.sign(signer);
await server.sendTransaction(tx);
```

## 2. Deposit

`deposit` pulls USDC from the caller, mints tranche tokens at the current NAV, and updates vault accounting in one transaction. It returns the number of shares minted. The on-chain signature is:

```rust
deposit(user: Address, vault_id: u32, kind: u32, amount: i128) -> i128
```

```ts
import {
  TransactionBuilder, BASE_FEE, Networks, Address, nativeToScVal, scValToNative,
} from '@stellar/stellar-sdk';

const TrancheKind = { Prime: 0, Core: 1, Alpha: 2 } as const;

async function deposit(vaultId: number, kind: number, amountUsdc: bigint) {
  const account = await server.getAccount(signer.publicKey());

  const tx = new TransactionBuilder(account, {
    fee: BASE_FEE,
    networkPassphrase: Networks.TESTNET,
  })
    .addOperation(
      contract.call(
        'deposit',
        new Address(signer.publicKey()).toScVal(),     // user (signs + authorizes)
        nativeToScVal(vaultId, { type: 'u32' }),       // vault_id
        nativeToScVal(kind, { type: 'u32' }),          // kind (TrancheKind)
        nativeToScVal(amountUsdc, { type: 'i128' }),   // amount (7 decimals)
      ),
    )
    .setTimeout(30)
    .build();

  const prepared = await server.prepareTransaction(tx);
  prepared.sign(signer);
  return server.sendTransaction(prepared);
}

// Deposit 1,000 USDC into Core (1_000 * 1e7)
await deposit(0, TrancheKind.Core, 10_000_000_000n);
```

<Note>
  `user.require_auth()` runs inside the contract, so the `user` address must sign the transaction. With Freighter, the connected wallet is both the source account and the authorizing address.
</Note>

## 3. Read your position

Tranche tokens are standard Soroban tokens. Read your `pCORE` balance from the tranche token contract, and read NAV from the `Tranche` struct via `get_tranche`:

```ts
// NAV per share is Tranche.nav_per_share_q — a Q64.64 fixed-point i128.
const tranche = await readContract(PRISM_CORE, 'get_tranche',
  nativeToScVal(0, { type: 'u32' }),
  nativeToScVal(TrancheKind.Core, { type: 'u32' }),
);

const nav = Number(tranche.nav_per_share_q) / 2 ** 64;  // human-readable ratio
const shares = await readTokenBalance(PTOKEN_CORE, signer.publicKey());
const positionUsdc = (Number(shares) / 1e7) * nav;
```

Your redemption value is always `shares × NAV`, regardless of what you paid on the secondary market.

## 4. Withdraw

`withdraw` burns tranche tokens and returns USDC at the current NAV (the payout amount is the return value):

```rust
withdraw(user: Address, vault_id: u32, kind: u32, shares: i128) -> i128
```

```ts
async function withdraw(vaultId: number, kind: number, shares: bigint) {
  const account = await server.getAccount(signer.publicKey());
  const tx = new TransactionBuilder(account, { fee: BASE_FEE, networkPassphrase: Networks.TESTNET })
    .addOperation(contract.call('withdraw',
      new Address(signer.publicKey()).toScVal(),   // user
      nativeToScVal(vaultId, { type: 'u32' }),     // vault_id
      nativeToScVal(kind, { type: 'u32' }),        // kind
      nativeToScVal(shares, { type: 'i128' }),     // shares to burn
    ))
    .setTimeout(30)
    .build();
  const prepared = await server.prepareTransaction(tx);
  prepared.sign(signer);
  return server.sendTransaction(prepared);
}
```

Withdrawal is allowed while the vault is `Active` or `Defaulted`, so LPs can exit a written-down tranche. Redeeming a wiped tranche (NAV 0) burns the shares and pays 0 — that is intentional.

## Error handling

Failed simulations and submissions carry the `prism_core` error code. Map it to a message before showing it to a user:

```ts
const PRISM_ERRORS: Record<number, string> = {
  1:  'This vault is not active.',
  2:  'Protocol is paused — deposits and withdrawals are temporarily disabled.',
  14: 'This tranche has been wiped; deposits are blocked while worthless shares remain outstanding.',
};

function explain(code: number) {
  return PRISM_ERRORS[code] ?? `Contract error ${code}`;
}
```

See the full table in [SDK & Bindings](/developer/sdk#error-codes).
