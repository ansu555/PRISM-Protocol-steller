# Network

PRISM runs on **Stellar Testnet** today. This page covers everything you need to wire a client to the network: endpoints, RPC selection, faucets, local development, and the path to mainnet.

## Endpoints

| | |
| --- | --- |
| Network | Stellar Testnet |
| Soroban RPC | `https://soroban-testnet.stellar.org` |
| Horizon | `https://horizon-testnet.stellar.org` |
| Network passphrase | `Test SDF Network ; September 2015` |
| Explorer | [stellar.expert — testnet](https://stellar.expert/explorer/testnet) |
| XLM faucet | [friendbot.stellar.org](https://friendbot.stellar.org) |

Stellar identifies a network by its **passphrase**, not a numeric chain ID. Always pass `Networks.TESTNET` (which equals the passphrase above) when building transactions — signing against the wrong passphrase produces an invalid signature.

## Soroban RPC vs. Horizon

Stellar exposes two APIs, and PRISM uses both:

| API | Use it for |
| --- | --- |
| **Soroban RPC** | Simulating, preparing, and submitting smart-contract transactions; reading contract state; streaming contract `getEvents`. |
| **Horizon** | Account balances, trustlines, payment history, and other classic-ledger queries. |

```ts
import { rpc as SorobanRpc, Horizon } from '@stellar/stellar-sdk';

const soroban = new SorobanRpc.Server('https://soroban-testnet.stellar.org');
const horizon = new Horizon.Server('https://horizon-testnet.stellar.org');
```

## RPC selection

| Endpoint | Pros | Cons | Best for |
| --- | --- | --- | --- |
| Public (`soroban-testnet.stellar.org`) | Free, no setup | Shared, rate-limited under load | Development, CI |
| Self-hosted / commercial RPC | Higher limits, reliable | Requires setup or sign-up | Production frontends, indexers |

The PRISM frontend reads its endpoint from `NEXT_PUBLIC_SOROBAN_RPC_URL` and defaults to the public testnet RPC.

## Finality

Stellar has a single, fast finality model — there are no commitment levels to choose. A ledger closes about every **5 seconds** via the Stellar Consensus Protocol, and once a transaction is in a closed ledger it is final (no rollbacks). After `sendTransaction`, poll `getTransaction(hash)` until the status is `SUCCESS`.

## Funding accounts

### XLM (fees)

Every account needs a small XLM balance to pay transaction fees and meet reserves. On testnet, fund an account with Friendbot:

```bash
curl "https://friendbot.stellar.org/?addr=<G-ADDRESS>"
```

### USDC

The testnet build uses a mintable test USDC (`PTUSDC`) whose issuer is the deployer, so scripts can fund accounts freely. An account must first hold a **trustline** to the asset (see [Deposit into a Tranche](/developer/invest#1-hold-usdc)); the deployer then sends USDC with a standard payment.

## Local development

Build and test the contract with cargo and the [Stellar CLI](https://developers.stellar.org/docs/tools/cli) — there is no separate validator process to manage:

```bash
cd soroban

# Build the Soroban WASM artifact
stellar contract build --package prism-core

# Run the on-chain test harness (in-process, deterministic)
cargo test -p prism-core
```

The Rust tests use Soroban's `Env` test harness, which runs the contract in-process — instant, deterministic, and offline. See [Test Suite](/developer/test-suite) for the full workflow. To exercise a live deploy, point the CLI at testnet with `--network testnet`.

## Common errors

| Symptom | Cause | Fix |
| --- | --- | --- |
| `tx_insufficient_fee` | Resource fee too low | Use `prepareTransaction` — it sets the Soroban resource fee |
| `op_no_trust` | Account has no USDC trustline | Add a `changeTrust` operation first |
| `Error(Contract, #2)` | Protocol is paused (`VaultPaused`) | Wait for unpause; reads still work |
| `txBadAuth` | Wrong network passphrase or signer | Sign with `Networks.TESTNET` and the authorizing address |

## Mainnet

Moving to mainnet means: swap the passphrase to `Networks.PUBLIC`, point at a mainnet Soroban RPC, use Circle's official USDC SAC, and rotate the admin key to a multisig. Set `NEXT_PUBLIC_STELLAR_NETWORK=mainnet` to flip the frontend's address registry. Testnet keys and assets have no monetary value — never reuse them on mainnet.
