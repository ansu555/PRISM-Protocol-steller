# Addresses

All addresses below are on **Stellar Testnet**. Mainnet contract IDs are published separately once the production deployment is live.

Soroban contract IDs are 56-character StrKeys that begin with `C`. They are derived deterministically from the contract's WASM hash plus a salt at deploy time. Account addresses begin with `G`.

## State & events

PRISM exposes activity primarily through its **getters** — clients poll `get_vault`, `get_tranche`, and `get_loan` to track balances, NAV, and loan state. The contract publishes a `seed_pool` contract event when liquidity is seeded into Soroswap; standard USDC and tranche-token transfers are observable as token-contract events. Index any of these from Soroban RPC `getEvents`.

## Inspect any address

With the [Stellar CLI](https://developers.stellar.org/docs/tools/cli):

```bash
# Read the live protocol config
stellar contract invoke \
  --id CCULBBT4PA64GWXSKT4G7HOYQ4RXRNYY2JP5MZ2G73VKRFAJ6CHB3RZK \
  --network testnet \
  -- get_config

# Read vault 0
stellar contract invoke \
  --id CCULBBT4PA64GWXSKT4G7HOYQ4RXRNYY2JP5MZ2G73VKRFAJ6CHB3RZK \
  --network testnet \
  -- get_vault --vault_id 0
```

Or open [stellar.expert (testnet)](https://stellar.expert/explorer/testnet) and paste any address from the tables above.
